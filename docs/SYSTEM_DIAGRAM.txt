BACKEND API (Flask + Python)
┌──────────────────────────────────────────────────────────────────────┐
│ Port: 5000                                                            │
│ File: api_server.py                                                  │
│                                                                       │
│ Endpoints:                                                            │
│  GET  /api/health       (Health check)                               │
│  POST /api/chat         (Main chat endpoint)                         │
│  GET  /api/init         (Initialize services)                        │
│  GET  /api/models       (Model information)                          │
│                                                                       │
│ Processing Pipeline:                                                 │
│  1. Receive message JSON                                             │
│  2. Check semantic cache (0.90 threshold)                            │
│  3. If cache miss: retrieve RAG context                              │
│  4. Build prompt with context                                        │
│  5. Call Gemini API (gemini-2.5-flash)                              │
│  6. Cache response for future use                                    │
│  7. Return JSON response                                             │
│                                                                       │
│ Dependencies:                                                         │
│  ✅ Flask (REST API)                                                 │
│  ✅ Flask-CORS (Cross-origin)                                        │
│  ✅ SentenceTransformers (Embeddings)                                │
│  ✅ Faster Whisper (Speech-to-text)                                  │
│  ✅ Google Gemini API (LLM)                                          │
│  ✅ Custom modules (cache, chunking, retrieval)                      │
└──────────────────────────────────────────────────────────────────────┘

KNOWLEDGE BASE & CACHE
┌──────────────────────────────────────────────────────────────────────┐
│ Data/                                                                 │
│  ├─ admission_procedures.json   (Admission info)                     │

│  ├─ departments_info.json       (Programs & courses)                 │
│  └─ facilities_detailed.json    (Campus facilities)                  │
│                                                                       │
│ Processing:                                                           │
│  Raw JSON (4 files)                                                  │
│    ↓                                                                  │
│  Chunking (95 semantic chunks)                                       │
│    ↓                                                                  │
│  Embeddings (all-MiniLM-L6-v2)                                      │
│    ↓                                                                  │
│  Stored in memory (built on startup)                                │

│ Cache:                                                                │
│  cache.json (Persistent file)                                       │
│  - Stores question → response mappings                              │
│  - Similarity threshold: 0.90                                        │
│  - Max 800 entries (LRU eviction)                                    │
│  - Fast retrieval: <100ms                                           │

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

"API Server not connected":
  → Make sure: python api_server.py is running
  → Check: both servers started before opening browser

Port 5000 or 3000 already in use:
  → Kill the process:
  → taskkill /PID <id> /F

npm not found:
  → Install Node.js: https://nodejs.org

Flask errors:
  → Make sure venv is activated
  → pip install flask flask-cors

API returns error 503:
  → Gemini API is busy
  → Wait a minute and try again
